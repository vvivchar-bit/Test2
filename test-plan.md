# План тестування програмної системи «Готель»

## Мета тестування

Мета тестування — перевірити роботу основних функцій програмної системи «Готель»: перегляд номерів, бронювання, перевірку введених даних, збереження заявки, пошук і фільтрацію номерів, а також відкриття системи у браузері.

## Об'єкт тестування

Навчальний вебпрототип **«Програмна система Готель»**.

## Інструмент тестування

Для керування тест-кейсами та результатами використано середовище **ReQtest**.

## Тип тестування

- System test;
- ручне функціональне тестування;
- перевірка інтерфейсу в браузері.

## Тестовий запуск

Назва тестового запуску в ReQtest:

```text
TR-02 System-Hotel functional testing
```

## Функції, що перевіряються

1. Відображення списку номерів.
2. Створення заявки на бронювання.
3. Перевірка введених даних.
4. Збереження заявки у списку бронювань.
5. Пошук і фільтрація номерів.
6. Відкриття системи у браузері та зручність інтерфейсу.

## Критерій успішності

Тест вважається пройденим, якщо фактичний результат відповідає очікуваному, а в ReQtest встановлено результат **OK**.
